## 🔧 Quick Configuration (REQUIRED!)

### ⚠️ IMPORTANT
You MUST configure ADMIN_WALLET_ADDRESS for the system to work!

### How to Configure

1. Open file: `frontend/js/auth.js`
2. Find line 5:
   ```javascript
   const ADMIN_WALLET_ADDRESS = '0x1234567890123456789012345678901234567890';
   ```
3. Replace with your admin wallet address:
   ```javascript
   const ADMIN_WALLET_ADDRESS = '0x90F8bf6A479f320ead074411a4B0e7944Ea8c9C1';
   ```

**Where to get the address?**
- From Ganache (shows on startup)
- From MetaMask (click account)
- Use any Ethereum address you own

### Ganache Users
Use Account 0: `0x90F8bf6A479f320ead074411a4B0e7944Ea8c9C1`

## 📊 System Behavior

### Admin Wallet
```
Login with: 0x90F8bf... (matches ADMIN_WALLET_ADDRESS)
   ↓
Role: ADMIN
   ↓
Redirect: /admin/dashboard.html
   ↓
Access: Full admin features
```

### Student Wallet
```

```

## ❓ Common Questions

**Q: Do I need to change the smart contract?**
A: No! Smart contract is unchanged.

**Q: Can I have multiple admins?**
A: Currently no, but can be added in future (requires code change).

**Q: What if I forget the admin wallet?**
A: Change ADMIN_WALLET_ADDRESS in auth.js to new wallet.

**Q: Is this secure for production?**
A: Good for dev/testnet. For production, consider backend auth.

**Q: Can I still use username/password?**
A: No, replaced completely with address-based auth.

## 🎓 How It Works (Technical)

**Old System (Username/Password)**
```
User Input (username, password)
  ↓
Backend Validation
  ↓
If match → Create session
  ↓
Redirect based on role
```

**New System (MetaMask Wallet)**
```
User clicks "Connect MetaMask"
  ↓
MetaMask popup shows
  ↓
User selects account & approves
  ↓
Frontend receives wallet address
  ↓
Compare with ADMIN_WALLET_ADDRESS
  ↓
Assign role (ADMIN or STUDENT)
  ↓
Create session with wallet address
  ↓
Redirect based on role
```
